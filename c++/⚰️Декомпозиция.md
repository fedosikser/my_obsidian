# C++ шпаргалка: декомпозиция кода

> [!tip] Главное правило
> **`.h` — "что" делает модуль (интерфейс)**  
> **`.cpp` — "как" это работает (реализация)**

---

## 📁 Базовая структура

```cpp
// MyClass.h
#pragma once

class MyClass {
public:
    MyClass();
    void doSomething(int value);
    int getResult() const;
    
private:
    int m_result;
};

```

```cpp
// MyClass.cpp
#include "MyClass.h"

MyClass::MyClass() : m_result(0) {}

void MyClass::doSomething(int value) {
    m_result = value * 2;
}

int MyClass::getResult() const {
    return m_result;
}
```

## 🗂️ Структура проекта

```
project/
├── include/          # публичные .h
│   └── myproject/
│       ├── MyClass.h
│       └── Utils.h
├── src/              # .cpp файлы
│   ├── MyClass.cpp
│   ├── Utils.cpp
│   └── main.cpp
└── CMakeLists.txt
```

## 🎯 Когда что использовать

| Сценарий | Решение |
|----------|---------|
| Обычный класс | `.h` + `.cpp` |
| Маленький класс (3-5 методов) | всё в `.h` (inline) |
| Шаблоны | только `.h` |
| Статические утилиты | `namespace` в `.h`, реализация в `.cpp` |
| Глобальные константы | `extern` в `.h`, определение в одном `.cpp` |

## ⚠️ Частые ошибки

```cpp
// ❌ НЕЛЬЗЯ
#include "MyClass.cpp"  // никогда!
using namespace std;    // в .h файлах
```

```cpp
// ✅ МОЖНО
#include "MyClass.h"     // только .h
std::vector<int> data;   // явный std::
```

## 🚀 Pimpl идиома (для больших проектов)

```cpp
// MyClass.h
class MyClass {
public:
    MyClass();
    ~MyClass();
    void doSomething();
    
private:
    class Impl;
    std::unique_ptr<Impl> pImpl;
};

// MyClass.cpp
class MyClass::Impl {
public:
    void doSomething() { /* реальная работа */ }
};

MyClass::MyClass() : pImpl(std::make_unique<Impl>()) {}
MyClass::~MyClass() = default;
void MyClass::doSomething() { pImpl->doSomething(); }
```
