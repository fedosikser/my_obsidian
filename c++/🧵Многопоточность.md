# 🧵 C++ шпаргалка: многопоточность

> [!tip] Главное правило
> **Разделяемые данные → защита мьютексом**  
> **Если не защитили → data race → undefined behavior**

---

## 📊 Основные компоненты

| Компонент | Назначение | Хедер |
|-----------|------------|-------|
| `std::thread` | запуск и управление потоком | `<thread>` |
| `std::mutex` | взаимное исключение (блокировка) | `<mutex>` |
| `std::lock_guard` | RAII для мьютекса (автоблокировка) | `<mutex>` |
| `std::unique_lock` | гибкая блокировка (можно отложить) | `<mutex>` |
| `std::condition_variable` | ожидание события | `<condition_variable>` |
| `std::atomic` | атомарные операции без мьютексов | `<atomic>` |
| `std::future`/`std::promise` | результат из потока | `<future>` |
| `std::async` | асинхронный запуск | `<future>` |

## 🚀 Запуск потоков

### Базовый запуск

```cpp
#include <thread>
#include <iostream>

void hello() {
    std::cout << "Привет из потока!" << std::endl;
}

int main() {
    std::thread t(hello);  // создаём и запускаем
    t.join();              // ждём завершения
    return 0;
}
```

### С параметрами

```cpp
void process(int x, const std::string& s) {
    std::cout << x << ": " << s << std::endl;
}

std::thread t(process, 42, "hello");  // передаём аргументы
t.join();
```

### Лямбда-функции

```cpp
std::thread t([](int a, int b) {
    return a + b;
}, 5, 10);
t.join();
```

## 🔒 Защита данных (мьютексы)

### Проблема: гонка данных

```cpp
int counter = 0;  // ❌ разделяемая переменная

void increment() {
    for (int i = 0; i < 100000; ++i) {
        ++counter;  // НЕ атомарно! data race!
    }
}
```

### Решение: мьютекс

```cpp
#include <mutex>

int counter = 0;
std::mutex mtx;

void increment() {
    for (int i = 0; i < 100000; ++i) {
        mtx.lock();      // захватываем
        ++counter;
        mtx.unlock();    // освобождаем
    }
}

// ✅ ЛУЧШЕ: lock_guard (RAII)
void incrementSafe() {
    for (int i = 0; i < 100000; ++i) {
        std::lock_guard<std::mutex> lock(mtx);
        ++counter;  // мьютекс автоматически освободится
    }
}
```

---

## 🛡️ Виды блокировок

| Тип | Особенности | Когда использовать |
|-----|-------------|-------------------|
| `lock_guard` | самый простой, нельзя разблокировать досрочно | 90% случаев |
| `unique_lock` | можно `lock()`/`unlock()`, отложенная блокировка | условные переменные, перемещение |
| `scoped_lock` | блокировка нескольких мьютексов сразу | чтобы избежать deadlock |

```cpp
std::mutex m1, m2;

// lock_guard
std::lock_guard<std::mutex> lock(m1);

// unique_lock (можно отложить)
std::unique_lock<std::mutex> lock(m1, std::defer_lock);
lock.lock();   // потом захватить
// ...
lock.unlock(); // досрочно освободить

// scoped_lock (несколько мьютексов без deadlock)
std::scoped_lock lock(m1, m2);  // захватывает оба атомарно
```

## ⏳ Ожидание событий (condition_variable)

```cpp
#include <condition_variable>

std::mutex mtx;
std::condition_variable cv;
bool ready = false;

// Поток-производитель
void producer() {
    std::this_thread::sleep_for(std::chrono::seconds(1));
    
    {
        std::lock_guard<std::mutex> lock(mtx);
        ready = true;
    }
    cv.notify_one();  // будим один поток
}

// Поток-потребитель
void consumer() {
    std::unique_lock<std::mutex> lock(mtx);
    cv.wait(lock, []{ return ready; });  // ждём condition
    std::cout << "Проснулись! ready = " << ready << std::endl;
}
```

> [!warning] Важно
> `wait()` может просыпаться **ложно** (spurious wakeup). Всегда используйте **предикат** (лямбду)!

## ⚛️ Атомарные операции (без мьютексов)

```cpp
#include <atomic>

std::atomic<int> counter = 0;  // атомарный счётчик

void increment() {
    for (int i = 0; i < 100000; ++i) {
        counter++;  // атомарно, без мьютекса!
    }
}

// Другие операции
std::atomic<bool> flag(false);
flag.store(true);
bool val = flag.load();

// Compare-and-swap
int expected = 42;
std::atomic<int> a(42);
a.compare_exchange_strong(expected, 100);  // если a == 42, станет 100
```

## 📦 async и future (простой способ)

```cpp
#include <future>
#include <chrono>

int slowComputation(int x) {
    std::this_thread::sleep_for(std::chrono::seconds(1));
    return x * 2;
}

int main() {
    // Запускаем асинхронно
    auto future = std::async(std::launch::async, slowComputation, 21);
    
    // Делаем что-то другое...
    std::cout << "Ждём результат..." << std::endl;
    
    // Получаем результат (блокируется, если не готов)
    int result = future.get();  // 42
    std::cout << "Результат: " << result << std::endl;
}
```

| Политика | Описание |
|----------|----------|
| `std::launch::async` | обязательно в новом потоке |
| `std::launch::deferred` | лениво, при вызове `.get()` в этом же потоке |
| `std::launch::async \| std::launch::deferred` | выбирает реализация (по умолчанию) |

## 🎯 Практические правила

| Ситуация | Решение |
|----------|---------|
| **Нужен счётчик** | `std::atomic<int>` |
| **Нужен флаг** | `std::atomic<bool>` |
| **Защита сложной структуры** | `std::mutex` + `lock_guard` |
| **Ожидание события** | `condition_variable` |
| **Запустить задачу и забыть** | `std::thread` с `detach()` |
| **Получить результат из потока** | `std::async` + `std::future` |
| **Пул потоков** | библиотека (например, `boost::asio` или ручная реализация) |

## ⚠️ Частые ошибки

```cpp
// ❌ 1. Забыли join() или detach()
std::thread t(work);
// t.~thread() -> terminate()!

// ✅
std::thread t(work);
t.join();  // или t.detach()

// ❌ 2. Data race
int shared = 0;
void bad() { shared++; }  // из нескольких потоков

// ✅
std::atomic<int> shared(0);

// ❌ 3. Deadlock (взаимная блокировка)
std::mutex a, b;
// поток 1: lock(a); lock(b);
// поток 2: lock(b); lock(a);

// ✅
std::scoped_lock lock(a, b);  // блокирует оба атомарно

// ❌ 4. Spurious wakeup
cv.wait(lock);  // без проверки условия

// ✅
cv.wait(lock, []{ return condition; });
```

## 🔄 Producer-Consumer (классический паттерн)

```cpp
#include <queue>
#include <thread>
#include <mutex>
#include <condition_variable>

template<typename T>
class ThreadSafeQueue {
private:
    std::queue<T> queue;
    mutable std::mutex mtx;
    std::condition_variable cv;
    
public:
    void push(T value) {
        {
            std::lock_guard<std::mutex> lock(mtx);
            queue.push(std::move(value));
        }
        cv.notify_one();
    }
    
    T pop() {
        std::unique_lock<std::mutex> lock(mtx);
        cv.wait(lock, [this]{ return !queue.empty(); });
        T value = std::move(queue.front());
        queue.pop();
        return value;
    }
    
    bool try_pop(T& value) {
        std::lock_guard<std::mutex> lock(mtx);
        if (queue.empty()) return false;
        value = std::move(queue.front());
        queue.pop();
        return true;
    }
};
```

## 📝 Быстрая памятка в терминале

```bash
# Скомпилировать с поддержкой многопоточности
g++ -std=c++17 -pthread program.cpp -o program

# Запустить (количество потоков зависит от кода)
./program

# Проверить утечки с ThreadSanitizer
g++ -std=c++17 -fsanitize=thread -g program.cpp -o program
```
