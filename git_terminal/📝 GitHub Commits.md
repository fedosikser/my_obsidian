## 🎯 **Основной формат (Conventional Commits)**

```text
<тип>[optional scope]: <описание>

[optional body]

[optional footer(s)]
```

> [!quote] Принцип
> Каждый коммит должен описывать **одно** логическое изменение. Не смешивай всё в кучу!

---

## 📋 **Типы коммитов**

| Тип | Эмодзи | Описание | Пример |
|-----|--------|----------|--------|
| `feat` | ✨ | Новая функциональность | `feat: добавить авторизацию` |
| `fix` | 🐛 | Исправление бага | `fix: исправить утечку памяти` |
| `docs` | 📚 | Документация | `docs: обновить README` |
| `style` | 💎 | Форматирование | `style: привести код к единому стилю` |
| `refactor` | ♻️ | Рефакторинг | `refactor: оптимизировать алгоритм` |
| `test` | 🧪 | Тесты | `test: добавить тесты для сортировки` |
| `perf` | ⚡ | Производительность | `perf: ускорить загрузку на 20%` |
| `chore` | 🔧 | Обслуживание | `chore: обновить зависимости` |
| `ci` | 👷 | CI/CD | `ci: обновить GitHub Actions` |
| `build` | 📦 | Сборка | `build: обновить CMakeLists.txt` |
| `revert` | ⏪ | Откат | `revert: откатить коммит abc123` |

---

## 🏷️ **Маркеры статуса**

| Маркер | Эмодзи | Значение | Пример |
|--------|--------|----------|--------|
| `[stable]` | ✅ | Стабильная версия | `feat: v2.0.0 [stable]` |
| `[beta]` | 🧪 | Тестируется | `feat: новый поиск [beta]` |
| `[alpha]` | 🚧 | Разработка | `feat: экспериментально [alpha]` |
| `[wip]` | 🔨 | В процессе | `feat: работа над модулем [wip]` |
| `[deprecated]` | ⚠️ | Устарело | `refactor: удалить старый API [deprecated]` |
| `[hotfix]` | 🔥 | Срочно | `fix: критический баг [hotfix]` |

## 🚀 **Команды терминала для коммитов**

### **Базовые команды**

```bash
# Создать коммит
git commit -m "feat: краткое описание"

# Создать коммит с подробным описанием
git commit -m "feat: добавить поиск" -m "Реализован полнотекстовый поиск по базе данных"

# Создать коммит с многострочным описанием
git commit -m "feat: поддержка WebP

- Добавлена библиотека libwebp
- Оптимизирована конвертация
- Добавлены тесты

Closes: #123
[beta]"
```

### **Просмотр и проверка**

```bash
# Посмотреть последние коммиты
git log --oneline -5

# Посмотреть изменения перед коммитом
git diff
git diff --staged

# Посмотреть статус
git status

# Посмотреть изменения в конкретном файле
git log -p filename.cpp
```

### **Исправление коммитов**

```bash
# Исправить последний коммит (сообщение или файлы)
git commit --amend -m "feat: новое описание"

# Добавить забытые файлы в последний коммит
git add forgotten-file.cpp
git commit --amend --no-edit

# Отменить последний коммит (сохранив изменения)
git reset --soft HEAD~1

# Отменить последний коммит (удалив изменения)
git reset --hard HEAD~1
```

### **Работа с областью (scope)**

```bash
# Коммит с областью
git commit -m "feat(auth): добавить Google авторизацию"

# Коммит с областью и статусом
git commit -m "feat(api): v2.0.0 стабильный релиз [stable]"
```

## 📝 **Примеры правильных коммитов**

### **Новая функция**

```bash
git commit -m "feat(search): полнотекстовый поиск

- Добавлен поиск по названию
- Добавлен поиск по содержимому
- Добавлена подсветка результатов

Тесты: test_search.cpp
Документация: docs/search.md
[beta]"
```

### **Исправление бага**

```bash
git commit -m "fix(parser): исправить краш при пустом вводе

Проблема: segmentation fault при передаче пустой строки
Причина: не проверялась длина строки перед парсингом
Решение: добавить проверку if (str.empty())

Fixes: #456
[stable]"
```

### **Релиз**

```bash
git commit -m "chore(release): v2.1.0

## Изменения
- ✨ Новый алгоритм сортировки
- 🐛 Исправлена утечка памяти
- ⚡ Ускорение на 30%

## Тестирование
- ✅ Unit tests (245 passed)
- ✅ Integration tests (56 passed)
- ✅ Performance tests

[stable]"
```

### **Рефакторинг**

```bash
git commit -m "refactor: оптимизировать алгоритм сортировки

- Замена пузырьковой сортировки на быструю
- Уменьшено потребление памяти
- Добавлены комментарии

Performance: O(n²) -> O(n log n)
[alpha]"
```

## 🎨 **Шаблоны для разных случаев**

### **Для новой фичи**

```bash
git commit -m "feat(${SCOPE}): ${DESCRIPTION}

## Что сделано
- ${CHANGE_1}
- ${CHANGE_2}

## Тестирование
- [ ] Unit tests
- [ ] Integration tests
- [ ] Manual testing

## Статус: ${STATUS}
Closes: #${ISSUE_NUMBER}"
```

### **Для багфикса**

```bash
git commit -m "fix(${SCOPE}): ${DESCRIPTION}

## Проблема
${PROBLEM_DESCRIPTION}

## Причина
${ROOT_CAUSE}

## Решение
${SOLUTION}

Fixes: #${ISSUE_NUMBER}
[${STATUS}]"
```

## ⚡ **Быстрые команды (шпаргалка)**

```bash
# Создать коммит
gc -m "feat: ..."

# Посмотреть историю
gl --oneline

# Исправить последний коммит
gca --amend

# Добавить файлы и создать коммит
ga . && gc -m "feat: ..."

# Посмотреть изменения
gd
gds

# Отменить последний коммит (сохранить изменения)
grh HEAD~1

# Отменить последний коммит (удалить изменения)
grh HEAD~1 --hard
```

> [!tip] Алиасы
> Добавь в `~/.gitconfig` для быстрого доступа:
> ```bash
> git config --global alias.ci commit
> git config --global alias.st status
> git config --global alias.co checkout
> git config --global alias.br branch
> git config --global alias.lg "log --oneline --graph"
> git config --global alias.last "log -1 HEAD"
> ```

## ✅ **Чек-лист перед коммитом**

- [ ] Заголовок до 50 символов
- [ ] Заголовок в императиве (добавить, исправить)
- [ ] Пустая строка после заголовка (если есть тело)
- [ ] Тело до 72 символов в строке
- [ ] Указан тип коммита (feat/fix/etc)
- [ ] Указана область (если нужно)
- [ ] Указан статус [stable/beta/alpha]
- [ ] Ссылки на issues (Closes #123)
- [ ] Все тесты проходят

---

## 🚫 **Чего НЕ делать**

> [!danger] Плохие примеры
> ❌ `git commit -m "fix bug"`
> ❌ `git commit -m "обнова"`
> ❌ `git commit -m "ы"`
> ❌ `git commit -m "исправлено"`
> ❌ `git commit -m "asdfghjk"`

> [!success] Золотое правило
> Всегда описывайте **ЧТО** и **ЗАЧЕМ** было изменено! Будущий ты скажет спасибо 🙏
