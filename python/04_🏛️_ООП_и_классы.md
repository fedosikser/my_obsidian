# 04_🏛️_ООП_и_классы

## Философия ООП в Python
- **Всё — объект** (даже функции, числа и типы)
- **Динамическая природа** — можно добавлять атрибуты на лету
- **Множественное наследование** (с осторожностью!)
- **Нет private/public/protected** — всё на доверии (по соглашению)
- **Магические методы** (`__method__`) управляют поведением

## Классы и объекты

### Простейший класс
```python
class Dog:
    """Класс собаки"""
    pass  # пустой класс

# Создание объекта
my_dog = Dog()
my_dog.name = "Бобик"  # можно добавлять атрибуты на лету!
print(my_dog.name)  # Бобик
```

### Конструктор `__init__` и атрибуты
```python
class Dog:
    # Конструктор (вызывается при создании объекта)
    def __init__(self, name, age):
        self.name = name  # атрибут объекта
        self.age = age
    
    # Метод
    def bark(self):
        print(f"{self.name} говорит Гав!")
    
    def get_human_age(self):
        """Собачий возраст в человеческих годах"""
        return self.age * 7

# Создание объекта
my_dog = Dog("Бобик", 3)
print(my_dog.name)        # Бобик
print(my_dog.age)         # 3
my_dog.bark()             # Бобик говорит Гав!
print(my_dog.get_human_age())  # 21
```

### Атрибуты класса vs атрибуты объекта
```python
class Dog:
    # Атрибут класса (общий для всех объектов)
    species = "Canis familiaris"
    dog_count = 0  # счетчик собак
    
    def __init__(self, name):
        # Атрибут объекта (у каждого свой)
        self.name = name
        Dog.dog_count += 1  # обращаемся через класс!
    
    def __del__(self):
        Dog.dog_count -= 1

d1 = Dog("Бобик")
d2 = Dog("Шарик")

print(d1.name)          # Бобик (свой)
print(d1.species)       # Canis familiaris (общий)
print(Dog.species)      # Canis familiaris (через класс)
print(Dog.dog_count)    # 2

# Изменение атрибута класса
Dog.species = "Canis lupus"
print(d2.species)       # Canis lupus (изменилось у всех)
```

## Магические методы (dunder methods)

### Основные магические методы
```python
class Book:
    def __init__(self, title, author, pages):
        self.title = title
        self.author = author
        self.pages = pages
        self._current_page = 1
    
    # Для пользователей (str)
    def __str__(self):
        return f"'{self.title}' - {self.author}"
    
    # Для разработчиков (repr)
    def __repr__(self):
        return f"Book('{self.title}', '{self.author}', {self.pages})"
    
    # Длина
    def __len__(self):
        return self.pages
    
    # Проверка вхождения
    def __contains__(self, word):
        return word in self.title or word in self.author
    
    # Доступ по индексу
    def __getitem__(self, page_num):
        if 1 <= page_num <= self.pages:
            return f"Страница {page_num}"
        raise IndexError("Страница вне диапазона")
    
    # Итерация
    def __iter__(self):
        self._current_page = 1
        return self
    
    def __next__(self):
        if self._current_page <= self.pages:
            result = f"Стр. {self._current_page}"
            self._current_page += 1
            return result
        raise StopIteration

book = Book("1984", "Джордж Оруэлл", 328)

print(str(book))       # '1984' - Джордж Оруэлл
print(repr(book))      # Book('1984', 'Джордж Оруэлл', 328)
print(len(book))       # 328
print("Оруэлл" in book)  # True
print(book[50])        # Страница 50

# Итерация
for page in book:      # работает благодаря __iter__ и __next__
    print(page)
    if page == "Стр. 3":
        break
```

### Математические операции
```python
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"
    
    # Арифметика
    def __add__(self, other):      # self + other
        return Vector(self.x + other.x, self.y + other.y)
    
    def __sub__(self, other):      # self - other
        return Vector(self.x - other.x, self.y - other.y)
    
    def __mul__(self, scalar):     # self * scalar
        return Vector(self.x * scalar, self.y * scalar)
    
    def __rmul__(self, scalar):    # scalar * self
        return self.__mul__(scalar)
    
    def __truediv__(self, scalar): # self / scalar
        return Vector(self.x / scalar, self.y / scalar)
    
    # Унарные операции
    def __neg__(self):              # -self
        return Vector(-self.x, -self.y)
    
    def __abs__(self):              # abs(self)
        return (self.x**2 + self.y**2)**0.5
    
    # Сравнения
    def __eq__(self, other):        # self == other
        return self.x == other.x and self.y == other.y
    
    def __lt__(self, other):        # self < other (по длине)
        return abs(self) < abs(other)

v1 = Vector(3, 4)
v2 = Vector(1, 2)

print(v1 + v2)        # Vector(4, 6)
print(v1 * 3)         # Vector(9, 12)
print(3 * v1)         # Vector(9, 12) (благодаря __rmul__)
print(-v1)            # Vector(-3, -4)
print(abs(v1))        # 5.0
print(v1 == Vector(3, 4))  # True
print(v1 < v2)        # False (5.0 < 2.236? Нет)
```

### Полный список магических методов

| Категория | Методы |
|-----------|--------|
| **Создание/удаление** | `__new__`, `__init__`, `__del__` |
| **Строковое представление** | `__str__`, `__repr__`, `__format__` |
| **Контейнеры** | `__len__`, `__getitem__`, `__setitem__`, `__delitem__`, `__contains__` |
| **Арифметика** | `__add__`, `__sub__`, `__mul__`, `__truediv__`, `__floordiv__`, `__mod__`, `__pow__` |
| **Обратная арифметика** | `__radd__`, `__rsub__`, `__rmul__` (когда левый операнд не поддерживает операцию) |
| **Сравнения** | `__eq__`, `__ne__`, `__lt__`, `__le__`, `__gt__`, `__ge__` |
| **Преобразования** | `__int__`, `__float__`, `__bool__`, `__complex__` |
| **Вызов** | `__call__` (объект как функция) |
| **Контекстный менеджер** | `__enter__`, `__exit__` |
| **Итераторы** | `__iter__`, `__next__` |
| **Атрибуты** | `__getattr__`, `__setattr__`, `__delattr__`, `__getattribute__` |

## Наследование

### Базовое наследование
```python
class Animal:
    def __init__(self, name):
        self.name = name
    
    def speak(self):
        raise NotImplementedError("Дочерний класс должен реализовать этот метод")
    
    def move(self):
        return f"{self.name} двигается"

class Dog(Animal):
    def speak(self):
        return f"{self.name} говорит Гав!"
    
    def wag_tail(self):
        return f"{self.name} виляет хвостом"

class Cat(Animal):
    def speak(self):
        return f"{self.name} говорит Мяу!"
    
    def purr(self):
        return f"{self.name} мурлычет"

# Использование
animals = [Dog("Бобик"), Cat("Мурка")]

for animal in animals:
    print(animal.speak())      # Полиморфизм!
    print(animal.move())       # Унаследованный метод
    # print(animal.wag_tail())  # Ошибка! Только у Dog
```

### super() — вызов родительского метода
```python
class Vehicle:
    def __init__(self, brand, model):
        self.brand = brand
        self.model = model
        self.speed = 0
    
    def info(self):
        return f"{self.brand} {self.model}"

class Car(Vehicle):
    def __init__(self, brand, model, doors):
        super().__init__(brand, model)  # вызов родительского __init__
        self.doors = doors
    
    def info(self):
        # Расширяем родительский метод
        return f"{super().info()}, {self.doors} двери"
    
    def __str__(self):
        return self.info()

car = Car("Toyota", "Camry", 4)
print(car)  # Toyota Camry, 4 двери
```

### Проверка типов и принадлежности
```python
class Animal: pass
class Dog(Animal): pass
class Cat(Animal): pass

d = Dog()
c = Cat()

# isinstance() — проверка принадлежности к классу (учитывает наследование)
print(isinstance(d, Dog))     # True
print(isinstance(d, Animal))  # True (Dog наследник Animal)
print(isinstance(d, Cat))     # False

# issubclass() — проверка наследования
print(issubclass(Dog, Animal))  # True
print(issubclass(Cat, Dog))     # False

# type() — точный класс (без учета наследования)
print(type(d) is Dog)         # True
print(type(d) is Animal)      # False
```

## Инкапсуляция (или её отсутствие)

### Соглашения об именах
```python
class Person:
    def __init__(self, name, age, ssn):
        self.name = name           # публичный (public)
        self._age = age             # защищенный (protected) — по соглашению
        self.__ssn = ssn            # приватный (private) — name mangling
    
    def get_ssn(self):              # геттер
        return self.__ssn[-4:]       # показываем только последние 4 цифры
    
    def _internal_method(self):      # "защищенный" метод
        print("Это внутренний метод")
    
    def __very_private(self):        # "приватный" метод
        print("Очень приватно")

p = Person("Анна", 30, "123-45-6789")

print(p.name)           # Анна (доступно)
print(p._age)           # 30 (доступно, но "не принято")
# print(p.__ssn)         # ОШИБКА! AttributeError

# Но Python не останавливает, если очень нужно:
print(p._Person__ssn)   # 123-45-6789 (name mangling)

print(p.get_ssn())      # 6789 (правильный доступ)
```

### Свойства (properties) — контроль доступа
```python
class Circle:
    def __init__(self, radius):
        self._radius = radius
        self._diameter = radius * 2
    
    @property
    def radius(self):
        """Геттер для радиуса"""
        return self._radius
    
    @radius.setter
    def radius(self, value):
        """Сеттер с валидацией"""
        if value < 0:
            raise ValueError("Радиус не может быть отрицательным")
        self._radius = value
        self._diameter = value * 2  # обновляем зависимое поле
    
    @property
    def diameter(self):
        """Вычисляемое свойство (только для чтения)"""
        return self._diameter
    
    @property
    def area(self):
        """Вычисляемое свойство"""
        return 3.14159 * self._radius ** 2

c = Circle(5)
print(c.radius)        # 5 (геттер)
c.radius = 10          # сеттер
print(c.diameter)      # 20
print(c.area)          # 314.159
# c.area = 100         # ОШИБКА! нет сеттера
# c.diameter = 30      # ОШИБКА! нет сеттера
```

### Современный способ: `@dataclass` (Python 3.7+)
```python
from dataclasses import dataclass, field

@dataclass
class Person:
    name: str
    age: int
    email: str = field(default="не указан")
    _id: int = field(default=0, repr=False)  # не показывать в __repr__
    
    def __post_init__(self):
        """Вызывается после __init__"""
        if self.age < 0:
            raise ValueError("Возраст не может быть отрицательным")

# Автоматически получаем __init__, __repr__, __eq__ и др.
p1 = Person("Анна", 30)
p2 = Person("Анна", 30)

print(p1)              # Person(name='Анна', age=30, email='не указан')
print(p1 == p2)        # True (автоматическое сравнение полей)
```

## Множественное наследование

### Базовый пример
```python
class Flyer:
    def fly(self):
        return "Я лечу!"
    
    def move(self):
        return "Я лечу в небе"

class Swimmer:
    def swim(self):
        return "Я плыву!"
    
    def move(self):
        return "Я плыву в воде"

class Duck(Flyer, Swimmer):
    def move(self):
        # Явно выбираем, чей move использовать
        return Swimmer.move(self)

duck = Duck()
print(duck.fly())       # Я лечу!
print(duck.swim())      # Я плыву!
print(duck.move())      # Я плыву в воде
```

### MRO (Method Resolution Order) — порядок разрешения методов
```python
class A:
    def method(self):
        return "A"

class B(A):
    def method(self):
        return "B"

class C(A):
    def method(self):
        return "C"

class D(B, C):
    pass

d = D()
print(d.method())       # B (смотрит в порядке D -> B -> C -> A)

print(D.__mro__)        # Посмотреть порядок:
# (<class '__main__.D'>, <class '__main__.B'>, 
#  <class '__main__.C'>, <class '__main__.A'>, <class 'object'>)
```

### Проблема ромба и super()
```python
class Animal:
    def __init__(self, name):
        self.name = name
        print(f"Animal __init__: {name}")

class Mammal(Animal):
    def __init__(self, name, fur_color):
        super().__init__(name)  # вызывает Animal.__init__
        self.fur_color = fur_color
        print(f"Mammal __init__: {name}, {fur_color}")

class Bird(Animal):
    def __init__(self, name, wingspan):
        super().__init__(name)  # вызывает Animal.__init__
        self.wingspan = wingspan
        print(f"Bird __init__: {name}, {wingspan}")

class Bat(Mammal, Bird):
    def __init__(self, name, fur_color, wingspan):
        # super() идет по MRO: Bat -> Mammal -> Bird -> Animal
        super().__init__(name, fur_color)  # wingspan??
        self.wingspan = wingspan
        print(f"Bat __init__: {name}, {fur_color}, {wingspan}")

bat = Bat("Бэтмен", "черный", 50)
# Animal __init__: Бэтмен
# Bird __init__: Бэтмен, 50 (удивительно! Bird вызвался из Mammal!)
# Mammal __init__: Бэтмен, черный
# Bat __init__: Бэтмен, черный, 50
```

## Статические и классовые методы

### `@classmethod` — метод класса
```python
class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day
    
    def __repr__(self):
        return f"Date({self.year}, {self.month}, {self.day})"
    
    @classmethod
    def from_string(cls, date_str):
        """Альтернативный конструктор из строки 'YYYY-MM-DD'"""
        year, month, day = map(int, date_str.split('-'))
        return cls(year, month, day)  # создает объект класса
    
    @classmethod
    def today(cls):
        """Возвращает сегодняшнюю дату"""
        from datetime import date
        d = date.today()
        return cls(d.year, d.month, d.day)

d1 = Date(2024, 3, 12)
d2 = Date.from_string("2024-12-31")
d3 = Date.today()

print(d2)  # Date(2024, 12, 31)
```

### `@staticmethod` — статический метод
```python
class MathUtils:
    @staticmethod
    def is_prime(n):
        """Проверка, является ли число простым"""
        if n < 2:
            return False
        for i in range(2, int(n**0.5) + 1):
            if n % i == 0:
                return False
        return True
    
    @staticmethod
    def factorial(n):
        if n <= 1:
            return 1
        return n * MathUtils.factorial(n - 1)

# Можно вызывать без создания объекта
print(MathUtils.is_prime(17))    # True
print(MathUtils.factorial(5))    # 120
```

### Разница между методами
```python
class Example:
    def instance_method(self, x):
        """Обычный метод — работает с объектом"""
        print(f"instance: self={self}, x={x}")
    
    @classmethod
    def class_method(cls, x):
        """Метод класса — работает с классом"""
        print(f"class: cls={cls}, x={x}")
    
    @staticmethod
    def static_method(x):
        """Статический метод — нет self или cls"""
        print(f"static: x={x}")

e = Example()
e.instance_method(1)    # instance: self=<...>, x=1
Example.instance_method(e, 1)  # то же самое

Example.class_method(2)  # class: cls=<class '__main__.Example'>, x=2
e.class_method(2)        # то же самое (класс определяется по объекту)

Example.static_method(3) # static: x=3
e.static_method(3)       # static: x=3
```

## Магические методы для контейнеров

### `__getitem__`, `__setitem__`, `__delitem__`
```python
class ShoppingCart:
    def __init__(self):
        self._items = {}
    
    def __getitem__(self, key):
        """cart['apple']"""
        return self._items.get(key, 0)
    
    def __setitem__(self, key, value):
        """cart['apple'] = 5"""
        if value < 0:
            raise ValueError("Количество не может быть отрицательным")
        self._items[key] = value
    
    def __delitem__(self, key):
        """del cart['apple']"""
        del self._items[key]
    
    def __contains__(self, key):
        """'apple' in cart"""
        return key in self._items
    
    def __len__(self):
        return len(self._items)
    
    def __iter__(self):
        return iter(self._items.items())

cart = ShoppingCart()
cart['apple'] = 5
cart['banana'] = 3

print(cart['apple'])      # 5
print('apple' in cart)    # True
print(len(cart))          # 2
del cart['apple']

for item, qty in cart:    # итерация
    print(f"{item}: {qty}")
```

### `__call__` — объект как функция
```python
class Multiplier:
    def __init__(self, factor):
        self.factor = factor
    
    def __call__(self, x):
        return x * self.factor

double = Multiplier(2)
triple = Multiplier(3)

print(double(5))    # 10
print(triple(5))    # 15

# Применение: декораторы, коллбэки
numbers = [1, 2, 3, 4]
doubled = list(map(double, numbers))  # [2, 4, 6, 8]
```

## Контекстные менеджеры (`with`)

### Реализация через класс
```python
class File:
    def __init__(self, filename, mode):
        self.filename = filename
        self.mode = mode
        self.file = None
    
    def __enter__(self):
        """Что делать при входе в блок with"""
        self.file = open(self.filename, self.mode)
        return self.file  # то, что попадает в переменную after 'as'
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        """Что делать при выходе из блока with"""
        if self.file:
            self.file.close()
        # Обработка исключений, если нужно
        if exc_type is not None:
            print(f"Произошла ошибка: {exc_val}")
        # return True — подавить исключение
        # return False (или None) — пробросить исключение дальше
        return False

with File("test.txt", "w") as f:
    f.write("Hello, World!")
    # файл автоматически закроется после выхода из блока
```

### Контекстный менеджер через `contextlib`
```python
from contextlib import contextmanager

@contextmanager
def temporary_file(filename, mode):
    """Контекстный менеджер через генератор"""
    print("Открываем файл")
    f = open(filename, mode)
    try:
        yield f  # здесь выполняется блок with
    finally:
        print("Закрываем файл")
        f.close()

with temporary_file("test.txt", "w") as f:
    f.write("Hello!")
```

## Сравнение с C++

| Концепция | C++ | Python |
|-----------|-----|--------|
| Модификаторы доступа | public/private/protected | Соглашения (_, __) |
| Конструктор | Имя класса | `__init__` |
| Деструктор | ~Имя класса | `__del__` (не надежный) |
| Этот объект | this | self (явно) |
| Перегрузка операторов | operator+ | `__add__`, `__sub__` и т.д. |
| Виртуальные функции | virtual | Все методы виртуальные |
| Множественное наследование | Есть (с осторожностью) | Есть (с MRO) |
| Дружественные функции | friend | Нет |
| Статические члены | static | `@staticmethod`, `@classmethod` |
