#### Задание 3.2: Декоратор для кэширования
Напишите свой декоратор `@cache(max_size=100)`, который кэширует результаты функции. Декоратор должен:
- Хранить ограниченное количество результатов (LRU — Least Recently Used)
- Иметь метод `cache_info()` для получения статистики (попадания/промахи)
- Иметь метод `cache_clear()` для очистки кэша

**Пример:**
```python
@cache(max_size=3)

    if n < 2:
        return n
    return fib(n-1) + fib(n-2)

print(fib(10))      # вычисляется
print(fib(10))      # берется из кэша
print(fib.cache_info())  # CacheInfo(hits=1, misses=11, maxsize=3, currsize=3)
```

**Подсказка:** Используйте словарь для хранения результатов и `collections.OrderedDict` для LRU.

#### Задание 10.3: Декоратор для транзакций
Напишите декоратор `@transactional`, который обеспечивает атомарность операции. Если функция выбрасывает исключение, все изменения должны быть отменены.

**Пример:**
```python
class BankAccount:
    def __init__(self):
        self.balance = 0
        self._changes = []
    
    @transactional
    def transfer(self, to_account, amount):
        self.balance -= amount
        to_account.balance += amount
        if amount > self.balance:  # проверка после изменений
            raise ValueError("Недостаточно средств")
        self._changes.append(f"Transfer {amount}")
```

**Подсказка:** Используйте паттерн "команда" (Memento). Декоратор должен сохранять состояние до вызова и восстанавливать при ошибке.
